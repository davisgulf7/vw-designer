### What is OpenSimulator?

**Image of past Moment:**  **Downloads:**  Binaries Zip (0.9.3.0)  Binaries Tarball (0.9.3.0)  Source Zip (0.9.3.0)  Source Tarball (0.9.3.0)  Diva distro currently only available for older mono/.net4x versions (0.9.2.1) [sha1 hashes](http://www.opensimulator.org/wiki/Template:ReleaseInfo) Other downloads may be found at either the [Download](http://www.opensimulator.org/wiki/Download) page, or the [OpenSim Release Repository](http://opensimulator.org/dist)

OpenSimulator is an open source multi-platform, multi-user 3D application server. It can be used to create a virtual environment (or world) which can be accessed through a variety of clients, on multiple protocols. It also has an optional facility (the [Hypergrid](http://www.opensimulator.org/wiki/Hypergrid)) to allow users to visit other OpenSimulator installations across the web from their 'home' OpenSimulator installation. In this way, it is the basis of a nascent distributed Metaverse.

OpenSimulator allows virtual world developers to customize their worlds using the technologies they feel work best - we've designed the framework to be easily extensible. OpenSimulator is written in [C#](http://www.ecma-international.org/publications/standards/Ecma-334.htm), running on Windows, Unix-like machines and on several plaforms on [.NET](https://dotnet.microsoft.com/). If you want to know about our development history, see [History](http://www.opensimulator.org/wiki/History).

Out of the box, OpenSimulator can be used to simulate virtual environments similar to [Second Life™](http://www.secondlifegrid.net/), given that it supports the core of [SL's messaging protocol](http://wiki.secondlife.com/wiki/Protocol). However, OpenSimulator is not just a clone of the Second Life server platform. Rather, the project aims to enable innovative feature development for virtual environments and the Metaverse at large. So only a subset of viewers for Second Life do implement Opensimulator features, see [Compatible Viewers](http://www.opensimulator.org/wiki/Compatible_Viewers).

OpenSimulator is getting more stable over time but is still a high complex software system that can suffer various bugs and quirks; handle with care!

### Features

Supports online, multi-user 3D environments as small as 1 simulator or as large as thousands of simulators.

Supports 3D virtual spaces of variable size.

Supports multiple clients and protocols - access the same world at the same time via multiple protocols.

Supports realtime Physics Simulation, with multiple engine options.

Supports clients that create 3D content in real time.

Supports inworld scripting using LSL/OSSL.

Provides unlimited ability to customize virtual world applications through the use of [scene plugin modules](http://www.opensimulator.org/wiki/IRegionModule).

For a more extensive list, see the [Feature Matrix](http://www.opensimulator.org/wiki/Feature_Matrix).

### Participating in the OpenSimulator Community

OpenSimulator is an [open source](http://en.wikipedia.org/wiki/Open_source) project, and is powered by the community members that devote time and energy to the effort. There are many ways to participate and contribute to the community:

Participate via [IRC](http://www.opensimulator.org/wiki/IRC). There are channels for users and developers.

Participate via the [Mailing Lists](http://www.opensimulator.org/wiki/Mailing_Lists). There are mailing lists for OpenSimulator use and development, as well as broader topics such as education and the Hypergrid.

[Contribute to this wiki](http://www.opensimulator.org/wiki/WikiStructure), making the OpenSimulator documentation even better. Don't be afraid of making mistakes - they can be easily corrected.

Report [bugs](http://www.opensimulator.org/wiki/Bugs) or submit [patches](http://www.opensimulator.org/wiki/Submitting_code_to_OpenSim) via our [mantis bug tracker](http://opensimulator.org/mantis/). If you're submitting code, please read through the [Contributions Policy](http://www.opensimulator.org/wiki/Contributions_Policy) before starting.

Create an OpenSimulator related project hosted on the [Forge](http://forge.opensimulator.org/) or [elsewhere](http://www.opensimulator.org/wiki/Related_Software) on the web. In the forge there are over a dozen registered projects, and it's a great way to further extend the OpenSimulator community.

Participate to open content creation for OpenSimulator. More details at [Artist Home](http://www.opensimulator.org/wiki/Artist_Home).

Participate in the weekly [Office Hours](http://www.opensimulator.org/wiki/Office_Hours) for OpenSimulator development.
